> 目标：7.15 前产出可体验 Demo（初赛），8.09 前交付完整产品（复赛）。
> 工具：全程 TRAE IDE（编程式 vibe coding，适合复杂前端项目），关键节点保留 Session ID 与对话截图。

## 一、产品定位

- **一句话**：像做 PPT 一样"造"网页——零代码可视化 HTML 编辑器
- **核心闭环**：自由画布拖拽 → 智能吸附 + 规则推断转响应式 → 一键导出干净单文件 HTML（无锁定）
- **初赛聚焦场景**：落地页 / 个人主页 / 活动页（高频、轻量、易演示）
- **不做**：复杂多页站点、CMS、电商、企业级应用（复赛再评估）

## 二、差异化卖点（评审打分锚点）

| 卖点 | 对应评审维度 | 说明 |
|---|---|---|
| 智能吸附 + 规则推断响应式 | 技术可行性 + 创新性 | 纯本地算法，0 API 成本。吸附对齐 + 同行/同列/嵌套关系推断 → Flex/Grid + flex-wrap 断点 |
| 干净单文件 HTML 导出 | 场景落地价值 | 无依赖、无锁定、可直接部署，对标 Wix/Framer 短板 |
| PPT 式极简交互 | 创意创新性 | 自由画布 + 图层树 + 属性面板，学习成本接近 0 |
| 可选 AI Pro 功能 | AI 融合（加分项） | 整页生成/文案/配色，用参赛 Token 额度或免费 API，成本可控时再加，非核心 |

## 三、技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 框架 | React 18 + TypeScript | 生态成熟，TRAE 生成质量高 |
| 状态 | Zustand | 轻量，适合编辑器状态机 |
| 拖拽 | dnd-kit | 现代、可访问、支持自由画布 |
| 样式 | Tailwind CSS + 内联样式 | 编辑器 UI 用 Tailwind，画布元素用内联样式便于导出 |
| 富文本 | Tiptap（可选） | 文本组件内编辑 |
| 撤销/重做 | zundo（Zustand 中间件） | 编辑器必备 |
| 构建 | Vite | 快，产物干净 |
| 导出 | 自研 HTML 序列化器 | 将画布 AST → 单文件 HTML |

> ⚠️ 合规确认：TRAE IDE 本质是 VS Code 系，可正常 `npm install` 上述开源库。"不允许第三方工具"应指 AI 工具/开发工具，非开源库。**建议向官方 FAQ 再确认一次**，若不允许则需 TRAE 原生实现拖拽等能力（成本高）。

## 四、模块拆分

```
PageForge/
├─ canvas/            # 画布渲染 + 拖拽 + 选中/吸附对齐
├─ components/        # 内置组件库（标题/图片/按钮/卡片/表单/容器）
├─ layers/            # 图层树面板
├─ inspector/         # 属性面板（样式/布局关系）
├─ toolbar/           # 顶部工具栏 + 撤销重做 + 预览/导出
├─ store/             # Zustand 状态（文档模型 AST + 历史）
├─ layout-rules/      # 吸附 + 规则推断响应式（纯本地算法）
├─ ai/                # 可选 AI Pro（整页生成/文案/配色，复赛评估）
├─ export/            # AST → 单文件 HTML 序列化
└─ preview/           # 多端预览（桌面/平板/手机）
```

**文档模型（AST）**：树形结构，节点 = { id, type, props, style, children, layoutHint }。`layoutHint` 供规则引擎推断响应式布局（同行/同列/嵌套）。

## 五、核心难点与解法

### 难点 1：自由画布 → 响应式（招牌功能，0 API 成本）
- **问题**：绝对坐标无法自适应。
- **解法**：智能吸附 + 规则推断，纯本地算法（不依赖 LLM，0 成本）。
  - 拖拽时：吸附到网格、对齐线、其他元素边缘（参考 WPS/PPT 智能对齐）
  - 放下时：根据 x/y 重叠关系推断"同行 / 同列 / 嵌套"
  - 生成时：同行→`flex-row`、同列→`flex-col`、嵌套→`container`；窄屏 `flex-wrap` 自动换行
- **初赛实现**：吸附 + 规则引擎，覆盖 80% 布局场景，保底可用。
- **复赛增强**：规则优化 + 可选 AI 兜底（复杂场景调 LLM，用参赛 Token 额度）。

### 难点 2：干净 HTML 导出
- **解法**：自研序列化器，AST → 语义化 HTML + 内联 CSS（或 `<style>` 块），零运行时依赖，单文件可直接双击打开。

### 难点 3：编辑器状态管理
- **解法**：Zustand + zundo，文档模型不可变更新。

## 七、风险与应对

| 风险 | 概率 | 应对 |
|---|---|---|
| 规则推断响应式覆盖不足 | 中 | 初赛保 80% 高频场景，复杂布局提示用户手动调布局关系 |
| 编辑器工程量超预期 | 高 | 初赛砍功能保核心闭环，复赛补全 |
| "全程 TRAE" 约束开源库使用 | 低 | 先按可用开源库推进，同时向官方确认 |
| 部署/托管被算作第三方工具 | 低 | 导出静态 HTML，用 TRAE 内置或 Vercel 部署 |
| 创新性被质疑"又一个建站工具" | 中 | 强打"智能吸附+规则响应式+干净导出+无锁定"组合，路演讲透行业困局 |
| AI Pro 功能 Token 成本 | 低 | 非核心功能，仅在参赛 Token 额度内启用，超量则下线该功能 |
