### 1. Demo 简介

**是什么**：造页工坊 PageForge 是一款**浏览器内的可视化网页搭建工具**（Web 应用），打开即用，无需安装。

**面向谁**：不会写代码却需要做网页的人——内容创作者、自媒体运营、小微企业主、学生、求职者。

**核心功能**：

- 🎨 **自由画布拖拽搭建**：像做 PPT 一样拖拽 14 种组件（标题、图片、按钮、卡片、表单、导航栏等）自由摆放。内置智能吸附对齐（边缘吸附、中心吸附、等间距吸附），拖拽时自动显示蓝色辅助线，放下即对齐。

- 📥 **一键导入预设模板**：内置 9 套精心适配的模板（SaaS 落地页、作品集、简历等），一键加载到画布编辑。也支持粘贴 HTML 或上传文件导入（开源模板兼容性持续优化中）。

- 📤 **一键导出生产级 HTML**：导出为干净自包含的单文件 HTML，含响应式 CSS（桌面/平板/手机三层断点）、Google Fonts 国内镜像、零依赖交互运行时。可独立部署、无平台锁定。额外支持导出 PNG 和 PDF。

### 2. Demo 创作思路

**灵感来源**：身边很多同学和朋友需要做个人主页、作品集或活动页，但不会写代码。每次找我帮忙，我其实也没那么多时间。就想做一个工具，让他们自己拖一拖就能搞定。

**想解决的问题**：把"找外包→沟通→设计→开发→部署"的流程压缩为"拖拽→导出"，让完全不会写代码的人在几分钟内做出能直接部署的网页。

**为什么做这个方向**：大赛主题是"人人都可以创造"。市面上很多工具要么学习曲线陡（Webflow 要理解 Flexbox/Grid），要么平台锁定（Wix 导不出代码），要么只托管不导出（Framer）。我想做一个**尊重用户产出物所有权**的工具——导出的 HTML 完全属于你，不被任何平台绑定。

**最大挑战：修动效 Bug——agent 看不到画面，只能靠文字描述**

项目中几乎每个功能都是和 TRAE 对话数十轮才打磨出来的，但最难的还是修**操作中的动态 Bug**。比如：

- 拖拽组件到画布，松手后落点相对预览偏移
- 旋转后的图片拖拽时，预览"飞"到别处，松手后又对了
- 裁切框手柄吸附触发时，裁切框会跳一下

这些 Bug 的共同点是**截图拍到结果、拍不到过程**。社区又不能发视频。我只能靠纯文字描述——"按住左键拖，松手前预览会跳一下，位置偏差大概 80px"——让 agent 在完全看不到动效的情况下理解问题。

这种情况下 agent 经常"修错"——改了代码但 Bug 还在，因为它的理解和我的画面之间存在天然 gap。比如拖拽预览偏移这个 Bug，表面是一个问题，拆开后是四个独立根因叠加：modifier 返回值理解错误、落点计算用中心而非左上角、吸附偏移读取时机错误、预览样式与落点不一致。每个根因都是一轮一轮对话试出来的。

踩坑多了，我总结出几个和 agent 协作修动效 Bug 的技巧：

1. **拆成步骤**：不说"拖拽有问题"，而是"第一步 mousedown ✓，第二步移动 ✓，第三步松手 ✗ 停在右下 80px"
2. **用类比**：说"预览飞了"不如说"就像从元素左上角弹射出去，弹射距离和元素大小有关"
3. **先让 agent 读源码**：让 TRAE 分析 dnd-kit 的 `DragOverlay` 定位源码，搞清楚底层机制再修——很多时候 agent 修错是因为它不理解依赖库的行为
